# Tailwind CSS Dialog/Modal

Use our Tailwind CSS dialog (also known as modal) component to inform users about a task or important information that require decisions, or involves multiple tasks.

A dialog is a type of modal window with critical information which disable all app functionality when they appear and remains on screen until confirmed/dismissed.

See below our modal examples styled with Tailwind CSS that you can use for your web or mobile project.

You can initialize a new dialog by adding the <Code>data-dialog-target="`{dialog}`"</Code> data attribute to the trigger element, the <Code>data-dialog-backdrop="`{dialogName}`"</Code> to the backdrop element and the <br /> <Code>data-dialog="`{dialogName}`"</Code> to the dialog element itself and make sure that the values for those data attributes are the same.

You can also use the <Code>data-dialog-close="true"</Code> for the element that you want to close the dialog with when clicking.

<br />

## Custom Dialog Animation

You can modify the open/close state animation for the modal by adding the <Code>data-dialog-mount="`{opacity-100}`"</Code>, <Code>data-dialog-unmount="`{opacity-0}`"</Code> and <Code>data-dialog-transition="`{transition-opacity}`"</Code> data attributes to the dialog element. You can pass Tailwind CSS classnames for those data attributes for animating the dialog. Check more about animation data attributes for dialog <a className="hover:text-blue-500 font-medium" href="#dialog-data-attributes">here</a>.

<CodePreview
  id="custom-dialog-animation"
  link="dialog#custom-dialog-animation"
  component={<HTMLDialogExamples.DialogCustomAnimation/>}
>

## Dialog Trigger Data Attributes

<span id="dialog-trigger-data-attributes" className="scroll-mt-48" />
The following data attributes are available for dialog trigger element.

| Attribute            | Description                                                                                                                                                                  |
| -------------------- | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `data-dialog-target` | Set the dialog target element, it should be the same as the `data-dialog` <br /> and `data-dialog-backdrop` data attributes.                                                 |
| `data-dialog-close`  | It's an optional data attribute, you can set it for any element and it will turn the element <br /> to a close trigger for dialog when click event happened on that element. |

---

## Dialog Backdrop Data Attributes

<span id="dialog-backdrop-data-attributes" className="scroll-mt-48" />
The following data attributes are available for dialog backdrop element.

| Attribute                    | Description                                                                                                                  |
| ---------------------------- | ---------------------------------------------------------------------------------------------------------------------------- |
| `data-dialog-backdrop`       | Set the dialog backdrop element, it should be the same as the `data-dialog` <br /> and `data-dialog-target` data attributes. |
| `data-dialog-backdrop-close` | Sets whether to close dialog when clicking on it's backdrop or not.                                                          |

---

## Dialog Data Attributes

<span id="dialog-data-attributes" className="scroll-mt-48" />
The following data attributes are available for dialog element.

| Attribute                 | Description                                                                                     | Default                                                                                                       |
| ------------------------- | ----------------------------------------------------------------------------------------------- | ------------------------------------------------------------------------------------------------------------- |
| `data-dialog`             | Set the name of the dialog and reference it to the <br /> dialog trigger and backdrop elements. |                                                                                                               |
| `data-popover-mount`      | Set the classnaes that should be used when the <br /> dialog is visible.                        | <Code>opacity-1</Code><br /><Code>translate-y-0</Code>                                              |
| `data-popover-unmount`    | Set the classnaes that should be used when the <br /> dialog is hidden.                         | <Code>opacity-0</Code><br /><Code>-translate-y-14</Code><br /><Code>pointer-events-none</Code> |
| `data-popover-transition` | Set the classnaes that should be used for <br /> transition of the dialog.                      | <Code>transition-all</Code><br /><Code>duration-300</Code>                                          |

## Modal Best Practices for Developers

• Prioritize user experience by making modals unobtrusive and easy to dismiss.<br />
• Include clear and easily accessible close buttons, and allow users to close the modal by clicking outside it.<br />
• Maintain a consistent design language and layout across modals to create a familiar user experience.<br />
• Use subtle animations and transitions to open and close modals smoothly. Make sure that animations do not distract users.<br />
• Implement validation and error handling for form elements within the modal.<br />
