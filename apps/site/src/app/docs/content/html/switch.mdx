# Tailwind CSS Switch

Use our Tailwind CSS switch component to let users adjust settings on/off. The option that the switch controls, as well as the state it's in, should be made clear from the corresponding inline label.

Switch components are used in various scenarios in web development, including settings options, feature toggles, form inputs, interactive controls, accessibility options, and more.

Use the following examples to create a simple and easy-to-use switch component for your Tailwind CSS project.

## Disabled Switch

This example illustrates a switch component designed to appear disabled and non-interactive. The key features indicating its disabled state include the use of the disabled attribute on the input element, as well as Tailwind CSS classes `opacity-50` and `pointer-events-none` applied to the `input` to visually and functionally disable the entire switch component.

<ComponentPreview demo={<HTMLSwitch.DisabledSwitch />}>
```html
<input
  class="relative inline-block h-6 w-10 appearance-none rounded-full before:absolute before:left-0 before:top-0 before:inline-block before:h-full before:w-full before:rounded-full before:bg-surface before:transition-colors before:duration-200 before:ease-in after:absolute after:left-0.5 after:top-2/4 after:h-5 after:w-5 after:-translate-y-2/4 after:rounded-full after:bg-background after:transition-transform after:duration-200 after:ease-in checked:before:bg-primary checked:after:translate-x-[calc(100%-4px)] cursor-not-allowed opacity-50 dark:after:bg-white"
  type="checkbox"
  disabled
/>
```
</ComponentPreview>
